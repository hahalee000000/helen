# Helen v1.24 更新报告

**发布日期**: 2026-07-21
**核心特性**: resume_session 改进 — 可见性标记 + 调用树完整性 + 幂等恢复

---

## 执行摘要

v1.24 解决了 resume_session 和 restore_context 的核心设计问题，实现三约束同时满足：

| 约束 | 解决方案 |
|------|----------|
| **确保可见性** | `visible_to_invocation_ids` 标记对哪些调用者可见 |
| **保留调用树** | 保留原始 `invocation_id` 和 `parent_invocation_id` |
| **多次恢复有效** | UUID 去重（幂等性） |

---

## 新增特性

### 1. Message 数据结构扩展

```python
@dataclass
class Message:
    # ... 现有字段 ...
    invocation_id: str = ""              # 原始调用者（不变）
    parent_invocation_id: str = ""       # 调用树关系（不变）
    visible_to_invocation_ids: list[str] = field(default_factory=list)  # 新增
```

### 2. resume_session 改进

**新行为**：
- 返回 dict（包含 `status`, `imported_messages`, `skipped_duplicates`, `session_id`）
- 保留原始 `invocation_id`（调用树完整性）
- 添加当前 `invocation_id` 到 `visible_to_invocation_ids`（确保可见性）
- UUID 去重（幂等性）

**使用示例**：
```helen
let result = resume_session("session_abc")
if result.status == "ok" {
    print("导入: {result.imported_messages}, 跳过: {result.skipped_duplicates}")
}
```

### 3. restore_context 改进

- 保留原始 `invocation_id`、`parent_invocation_id`、`agent_name`
- 添加当前 `invocation_id` 到 `visible_to_invocation_ids`
- 保留原始 UUID

### 4. _history 过滤逻辑更新

```python
return [m for m in all_messages
        if (m.invocation_id == self._current_invocation_id or
            self._current_invocation_id in m.visible_to_invocation_ids)]
```

### Message 字段扩展

- ✅ 旧 Message（无此字段）自动填充空列表
- ✅ 序列化兼容（JSONL/SQLite 可忽略未知字段）

### resume_session 返回类型变更

**破坏性变更**：返回类型从 `bool` 变为 `dict`

```helen

// 新代码
if resume_session("session_abc").status == "ok" { ... }
```
